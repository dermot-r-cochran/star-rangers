# Redline Protocol: Variability Governance

When contingent harm rises faster than verified capability gain, command shifts from acceleration to containment.

## Operating Model

### Layer 1 — Mechanism
Track the chain:

**entropy → variability → risk**

If variability broadens, risk surface broadens. This is expected.

### Layer 2 — Constraint
Determine minimum viable variability for mission objectives:

- truth-seeking
- adaptation to unknown conditions
- creative synthesis under uncertainty

Variability below this threshold collapses function. Variability above it accumulates avoidable risk.

### Layer 3 — Optimization
Reduce risk while preserving capability:

- tighten reward functions
- constrain unsafe search trajectories
- add adversarial review checkpoints
- narrow exploration breadth when harm rate accelerates

### Layer 4 — Evaluation
Partition observed harm:

\[
H_{total}=H_{necessary}+H_{contingent}
\]

Escalate when contingent component dominates trendline.

## Redline Triggers

1. Recurring harmful states already documented in prior cycles.
2. Harm growth without corresponding increase in validated mission value.
3. Incentive loops rewarding novelty over reliability.

## Mandatory Actions Under Redline

1. Freeze open-ended generation channels.
2. Reduce search breadth and stochasticity windows.
3. Require pre-deployment adversarial checks.
4. Re-open only when contingent-harm slope returns below doctrine threshold.

## Standing Ethical Test

> What fraction of suffering is structurally required vs reducible?

Policy failure begins when this question is not measured.
